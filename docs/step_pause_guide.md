# 步骤暂停功能使用指南

## ✅ 新增功能

已为流程步骤添加完整的暂停和恢复功能，允许在执行过程中临时中断步骤。

## 📊 状态流转图

### 完整状态流转
```
待开始 (pending)
    ↓ 点击"开始执行"
进行中 (in_progress)
    ↓ 点击"暂停" ← 新增
已暂停 (paused)
    ↓ 点击"恢复执行" ← 新增
进行中 (in_progress)
    ↓ 点击"完成步骤"
已完成 (completed)
```

### 状态说明
- **待开始** (pending) - 灰色，尚未开始
- **进行中** (in_progress) - 黄色，正在执行
- **已暂停** (paused) - 紫色，临时中断 ← 新增
- **已完成** (completed) - 绿色，已完成

## 💡 使用场景

### 场景1: 临时中断执行
```
参演人员开始执行步骤
    ↓
遇到问题需要暂停

点击"暂停"按钮

    ↓

```

```
开始执行步骤
    ↓
需要等待其他系统就绪
    ↓
点击"暂停"按钮
    ↓
等待外部条件满足
    ↓
点击"恢复执行"按钮
    ↓

```
正在执行步骤A
    ↓

    ↓

    ↓

## 🔧 技术实现

### 后端API
```go
// 暂停步骤
POST /api/executions/:id/pause
Response: {"message": "execution paused"}

// 恢复步骤
POST /api/executions/:id/resume
Response: {"message": "execution resumed"}
```

### 前端调用
```typescript
// 暂停步骤
await drillService.pauseExecution(execution.id)

// 恢复步骤
await drillService.resumeExecution(execution.id)
```

## 📝 注意事项

### 1. 状态限制
- 只有"进行中"状态可以暂停
- 只有"已暂停"状态可以恢复
- 已完成的步骤不能暂停或恢复

### 2. 时间计算
- 暂停期间不计入超时时间
- 恢复后继续累计执行时间
- 总耗时从开始时间计算

### 3. 权限要求
- 所有用户都可以暂停和恢复步骤
- 暂停不影响其他步骤的执行
- 可以多次暂停和恢复

## 🆘 常见问题

### Q: 暂停按钮在哪里？
**A**: 点击正在执行的步骤卡片，Modal底部会显示"暂停"和"完成步骤"两个按钮。

### Q: 暂停后可以恢复吗？
**A**: 可以。点击已暂停的步骤卡片，Modal底部会显示"恢复执行"按钮。

### Q: 暂停会影响其他步骤吗？
**A**: 不会。暂停只影响当前步骤，其他步骤可以正常执行。

### Q: 可以多次暂停吗？
**A**: 可以。暂停后恢复，然后可以再次暂停，没有次数限制。

### Q: 暂停期间计入超时吗？
**A**: 不计入。暂停期间不会累计超时时间。
